1
📌 Introduction — 15 sec

"Mera project hai NoteGenie — ek Multi-Agent AI Study Companion. Iska idea ye hai ki students apne notes upload karte hain — PDF, audio recordings, ya YouTube video link — aur NoteGenie unhe automatically summarize karta hai, flashcards generate karta hai, quizzes banata hai, aur personalized study plan create karta hai — sab kuch AI agents ke through."

2
⚙️ Tech Stack — 20 sec

"Maine ye project Next.js 14 pe build kiya hai with React 18, TypeScript, aur Supabase as backend. AI ke liye maine dual AI provider architecture use kiya — OpenAI GPT-5-mini for complex tasks like summarization aur question generation, aur Groq (LLaMA 3.3 70B + Whisper) for free audio transcription aur text generation. Plus Google Gemini Vision API for handwriting OCR."

3
🤖 Multi-Agent Architecture — 25 sec

"Main highlight hai mera multi-agent system — 14 specialized AI agents hain. Jaise Summarizer Agent notes summarize karta hai, Concept Extractor key terms nikaalta hai, Flashcard Agent Q&A cards generate karta hai, Question Maker MCQ/short/true-false questions banata hai, Doubt Resolver chatbot ki tarah doubts solve karta hai with analogies aur step-by-step explanations. Ye agents pipeline mein chained hain — ek ka output dusre ka input banta hai."

4
🎨 Key Features — 20 sec

"Project mein YouTube video to notes — sirf link paste karo, transcript extract hota hai aur notes ban jate hain. Audio transcription with Groq Whisper API — voice recordings se clean notes. Interactive Concept Graph — Canvas API pe force-directed mind map. Spaced Repetition algorithm for smart flashcard review. Aur Google Calendar integration for study plan sync."

5
🔐 Auth & Database — 20 sec

"Authentication Supabase Auth se handle hota hai — Email/Password + Google OAuth. Database mein PostgreSQL with Row Level Security (RLS) — har user sirf apna data access kar sakta hai. 8 database tables hain — profiles, summaries, flashcards, concepts, questions, study_tasks, user_stats, aur generated_books. Backend pe database triggers automatically profile aur stats create karte hain on signup."

6
🏆 Challenges & Fun — 20 sec

"Challenges jo maine solve kiye — multi-provider AI orchestration (OpenAI + Groq + Gemini ko seamlessly switch karna), agent pipeline management (14 agents ko coordinate karna), aur Canvas-based graph rendering with force-directed layout. Plus ek fun easter egg hai — Upside Down Mode inspired by Stranger Things, jisme entire UI flip ho jata hai with thunder sound effect! 🎃"

Interview Questions & Answers

10 most common questions interviewers will ask
Q1What is NoteGenie and what problem does it solve?
NoteGenie ek AI-powered study companion hai jo students ki notes se learning materials automatically generate karta hai. Problem ye hai ki students ko notes likhne ke baad manually flashcards banana padta hai, quizzes dhundhne padte hain, study plan banana padta hai — ye sab time-consuming hai. NoteGenie ye sab 14 specialized AI agents ke through automate karta hai — upload karo, aur seconds mein summaries, flashcards, quizzes, concept maps, aur study plans ready.
Q2Why multi-agent architecture instead of a single AI call?
Single AI call se sab kaam ek prompt mein karna unreliable hota hai — output consistency kharab hoti hai aur token limit hit hota hai. Multi-agent approach mein har agent ek specialized task karta hai with its own optimized prompt. Summarizer Agent sirf summary banata hai, Concept Extractor sirf key terms nikalta hai, fir Flashcard Agent un concepts ko cards mein convert karta hai. Ye separation of concerns pattern hai — each agent is small, testable, and replaceable. Plus agents ko chain kar sakte hain: ek ka output dusre ka input.
Q3Why did you use 3 different AI providers?
Cost optimization aur best-fit matching. OpenAI GPT-5-mini sabse accurate hai text tasks ke liye — summarization, question generation. Groq ka LLaMA 3.3 70B model free hai aur fast hai — audio notes ki summary ke liye perfect. Groq ka Whisper Large V3 best-in-class free audio transcription hai. Google Gemini Vision API image understanding mein best hai — handwritten notes ka OCR aur highlighted text detection. Har provider apne specific task mein excel karta hai.
Q4How does YouTube to Notes work internally?
Maine youtubei.js library use ki hai jo YouTube ka internal API (Innertube) access karti hai. User URL paste karta hai, maine extractVideoId function se video ID nikaali, fir Innertube.create() se YouTube client banaya, getInfo() se video info li, aur getTranscript() se captions extract kiye. Captions ko combine karke ek full transcript banaya, fir Groq ke LLaMA model se usse structured notes mein convert kiya — with title, one-liner, detailed bullets, key timestamps, aur main topics.
Q5How does the spaced repetition algorithm work?
Maine standard spaced repetition intervals implement kiye — 1, 3, 7, 14, 30 days. Memory Retention Agent har flashcard ke timesReviewed aur lastReviewed fields check karta hai. Agar card 0 times reviewed hai aur 1 din ho gaya — review due. 1 time reviewed aur 3 din ho gaye — review due. Ye scientifically proven hai ki increasing intervals mein review karne se long-term memory mein better retention hoti hai. Cards ko priority se sort karta hoon — least reviewed first.
Q6Explain how Row Level Security (RLS) works in your project.
Supabase mein har table pe RLS enable hai. Maine policies define ki hain like "Users can view own summaries" — jo check karti hai ki auth.uid() (logged-in user ka ID) match kare user_id column se. Matlab agar User A logged in hai, wo sirf apne summaries, flashcards, concepts Access kar sakta hai — chahe direct SQL query bhi kare. Ye database level security hai, application level nahi — bahut strong. 8 tables pe total 24+ RLS policies hain.
Q7What was the biggest technical challenge?
Multi-provider AI orchestration sabse challenging tha. OpenAI, Groq, aur Gemini — teenon ke APIs alag hain, response formats alag hain, error handling alag hai. Maine wrapper functions banaye (callOpenAI, callGroq) jo unified interface provide karte hain. Plus JSON parsing — AI kabhi kabhi invalid JSON return karta hai, toh maine robust parseJSONResponse function banaya jo markdown code blocks handle karta hai, partial JSON extract karta hai, aur graceful fallbacks provide karta hai.
Q8How does the interactive Concept Graph work?
Maine Canvas API use karke force-directed graph implement kiya. AI agent concepts ko nodes aur unke relationships ko edges mein convert karta hai. Fir Canvas pe har frame physics simulation run hoti hai — nodes ek dusre ko repel karte hain (Coulomb's law), edges connected nodes ko attract karte hain (spring force), aur center gravity sab ko middle mein rakhti hai. User drag kar sakta hai nodes ko, zoom in/out kar sakta hai. Har node ka color uske level pe depend karta hai — 6 gradient colors hain.
Q9How do you handle the Upside Down Mode?
Ye Stranger Things inspired easter egg hai! Maine UpsideDownContext banaya jo React Context API use karta hai. Jab user toggle karta hai, 2.5 second ka flip animation CSS transform se play hota hai, saath mein thunder sound effect (Web Audio API). CSS mein entire body ko transform: rotate(180deg) + scaleX(-1) karta hoon, colors dark red/black theme mein change hote hain, aur fonts Creepster (horror font) mein switch hote hain. Separate upside-down.css file hai poore dark theme ke liye.
Q10What did you learn from this project?
Multi-agent AI systems architecture — 14 agents ko design aur coordinate karna. Multi-provider AI integration — OpenAI, Groq, Gemini ko ek project mein seamlessly use karna. Supabase with Row Level Security — production-grade database security. Canvas API — force-directed graph rendering from scratch. Audio processing pipelines — Whisper transcription se clean notes tak. Aur sabse important — prompt engineering, kyunki har agent ka output quality directly prompt pe depend karta hai.
