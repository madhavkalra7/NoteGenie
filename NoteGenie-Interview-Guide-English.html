2-Minute Speaking Script

⏱️ Total Time: ~2 minutes
1
📌 Introduction — 15 sec

"My project is NoteGenie — a Multi-Agent AI Study Companion. The idea is simple — students upload their notes as PDFs, audio recordings, or YouTube video links — and NoteGenie automatically summarizes them, generates flashcards, creates quizzes, and builds personalized study plans — all through a pipeline of specialized AI agents."

2
⚙️ Tech Stack — 20 sec

"I built this project on Next.js 14 with React 18, TypeScript, and Supabase as the backend. For AI, I implemented a dual AI provider architecture — OpenAI GPT-5-mini for complex tasks like summarization and question generation, and Groq (LLaMA 3.3 70B + Whisper) for free audio transcription and text generation. I also integrated Google Gemini Vision API for handwriting OCR."

3
🤖 Multi-Agent Architecture — 25 sec

"The main highlight is my multi-agent system — 14 specialized AI agents. For example, the Summarizer Agent creates structured summaries, the Concept Extractor pulls out key terms and definitions, the Flashcard Agent generates Q&A cards, the Question Maker creates MCQ, short answer, and true/false questions, and the Doubt Resolver works like a chatbot that solves doubts with analogies and step-by-step explanations. These agents are chained in a pipeline — one agent's output becomes the next agent's input."

4
🎨 Key Features — 20 sec

"The project includes YouTube video to notes — just paste a link, transcript is extracted and notes are generated. Audio transcription via Groq Whisper API — voice recordings converted to clean notes. An Interactive Concept Graph — a force-directed mind map built on the Canvas API. A Spaced Repetition algorithm for smart flashcard review scheduling. And Google Calendar integration to sync study plans."

5
🔐 Auth & Database — 20 sec

"Authentication is handled through Supabase Auth — supporting Email/Password and Google OAuth. The database uses PostgreSQL with Row Level Security (RLS) — each user can only access their own data. There are 8 database tables — profiles, summaries, flashcards, concepts, questions, study_tasks, user_stats, and generated_books. On the backend, database triggers automatically create profile and stats records on signup."

6
🏆 Challenges & Fun — 20 sec

"Key challenges I solved include multi-provider AI orchestration (seamlessly switching between OpenAI, Groq, and Gemini), agent pipeline management (coordinating 14 agents), and Canvas-based graph rendering with force-directed layout using physics simulation. Plus there's a fun easter egg — Upside Down Mode inspired by Stranger Things, where the entire UI flips with a thunder sound effect! 🎃"

Interview Questions & Answers

10 most common questions interviewers will ask
Q1What is NoteGenie and what problem does it solve?
NoteGenie is an AI-powered study companion that automatically generates learning materials from student notes. The problem it solves is that after taking notes, students have to manually create flashcards, search for quizzes, and build study plans — all of which is extremely time-consuming. NoteGenie automates all of this through 14 specialized AI agents — just upload your notes, and within seconds you get summaries, flashcards, quizzes, concept maps, and personalized study plans.
Q2Why multi-agent architecture instead of a single AI call?
Doing everything in a single AI call is unreliable — output consistency degrades and you hit token limits quickly. With a multi-agent approach, each agent handles one specialized task with its own optimized prompt. The Summarizer Agent only creates summaries, the Concept Extractor only pulls out key terms, then the Flashcard Agent converts those concepts into cards. This follows the separation of concerns design pattern — each agent is small, testable, and replaceable. Plus agents can be chained: one agent's output becomes the next agent's input.
Q3Why did you use 3 different AI providers?
Cost optimization and best-fit matching. OpenAI GPT-5-mini is the most accurate for text tasks like summarization and question generation. Groq's LLaMA 3.3 70B model is free and fast — perfect for audio note summaries. Groq's Whisper Large V3 is the best-in-class free audio transcription service. Google Gemini Vision API excels at image understanding — handwritten note OCR and highlighted text detection. Each provider excels at its specific task, so using all three gives the best results at the lowest cost.
Q4How does YouTube to Notes work internally?
I used the youtubei.js library which accesses YouTube's internal API (Innertube). The user pastes a URL, I extract the video ID using regex pattern matching, then create a YouTube client via Innertube.create(), fetch video info with getInfo(), and extract captions via getTranscript(). The caption segments are combined into a full transcript, which is then sent to Groq's LLaMA model to generate structured notes — including title, one-liner summary, detailed bullet points, key timestamps, and main topics.
Q5How does the spaced repetition algorithm work?
I implemented standard spaced repetition intervals — 1, 3, 7, 14, and 30 days. The Memory Retention Agent checks each flashcard's timesReviewed and lastReviewed fields. If a card has been reviewed 0 times and 1 day has passed — it's due for review. If reviewed once and 3 days have passed — due again. This is scientifically proven to improve long-term memory retention through increasing review intervals. Cards are sorted by priority — least reviewed first, then by last review date.
Q6Explain how Row Level Security (RLS) works in your project.
Every table in Supabase has RLS enabled. I defined policies like "Users can view own summaries" which checks that auth.uid() (the logged-in user's ID) matches the user_id column. This means if User A is logged in, they can only access their own summaries, flashcards, and concepts — even with direct SQL queries. This is database-level security, not application-level, making it extremely robust. There are 24+ RLS policies across 8 tables covering SELECT, INSERT, UPDATE, and DELETE operations.
Q7What was the biggest technical challenge?
Multi-provider AI orchestration was the most challenging. OpenAI, Groq, and Gemini all have different APIs, response formats, and error handling patterns. I built wrapper functions (callOpenAI, callGroq) that provide a unified interface. Additionally, JSON parsing was tricky — AI sometimes returns invalid JSON, so I built a robust parseJSONResponse function that handles markdown code blocks, extracts partial JSON, and provides graceful fallbacks. This ensures the app never crashes due to unexpected AI output.
Q8How does the interactive Concept Graph work?
I implemented a force-directed graph using the Canvas API. The AI agent converts concepts into nodes and their relationships into edges. Then on each animation frame, a physics simulation runs on the Canvas — nodes repel each other (Coulomb's law), edges attract connected nodes (spring force), and center gravity keeps everything in the middle. Users can drag nodes, zoom in/out, and click for details. Each node's color is determined by its level — there are 6 gradient color schemes for visual hierarchy.
Q9How do you handle the Upside Down Mode?
This is a Stranger Things–inspired easter egg! I created an UpsideDownContext using React's Context API. When the user toggles it, a 2.5-second flip animation plays via CSS transforms, accompanied by a thunder sound effect using the Web Audio API. The CSS applies transform: rotate(180deg) + scaleX(-1) to the entire body, colors shift to a dark red/black theme, and fonts switch to Creepster (a horror-style font). There's a dedicated upside-down.css file for the complete dark theme styling.
Q10What did you learn from this project?
I learned multi-agent AI system architecture — designing and coordinating 14 specialized agents. Multi-provider AI integration — using OpenAI, Groq, and Gemini seamlessly in one project. Supabase with Row Level Security — implementing production-grade database-level security. Canvas API — building force-directed graph rendering from scratch with physics simulation. Audio processing pipelines — from Whisper transcription to clean structured notes. And most importantly, prompt engineering — because every agent's output quality depends directly on how well the prompt is crafted.
